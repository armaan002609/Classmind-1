# Waiting Room

## What it does
Holds students in a secure pre-class queue before the teacher officially launches the live session. Shows student photos, admission requests, and plays a preparation video.

## Why it exists
It prevents students from accessing classroom features until the teacher is ready to moderate and teach.

## When it should be used
Active between session creation and the official 'Start Session' action.

## How to use it
1. Create a session to generate the 6-digit code.
2. Students who join using this code will enter the Waiting Room.
3. Review student photos and names in the queue.
4. Click 'Approve' to admit them or 'Reject' to deny entry.
5. Click 'Start Session' to go live and move everyone to the dashboard.

## Best practices
Enable 'Auto-Join' in settings if you want to skip manual approval for pre-registered students.

## Common mistakes
Leaving students in the waiting room for long periods without starting the session.

## Troubleshooting steps
If students cannot connect to the waiting room, check if the session code was typed correctly.
